# Skill File Reference

`rcli init` generates instruction files for AI coding tools via `--tool`:

| File | Tool | Location |
|------|------|----------|
| `SKILL.md` | Claude Code | `.claude/skills/rcli/SKILL.md` | `--tool claude` (default) |
| `AGENTS.md` | OpenAI Codex | Project root | `--tool codex` |
| `OPENCODE.md` | OpenCode | Project root | `--tool opencode` |

## Purpose

Instructs AI coding tools how to interact with the rcli database:
- Always use `--format json` for programmatic access
- Run `rcli init` if `.rcli/` doesn't exist
- Documents all commands with examples
- Provides workflow guidelines

All three files share the same core instructions. Only the framing differs:
- **SKILL.md** has YAML frontmatter (`name`, `description`, `user-invocable`) for Claude Code's skill discovery
- **AGENTS.md** and **OPENCODE.md** use a plain markdown header

## Merge Behavior

For `AGENTS.md` and `OPENCODE.md`, if the file already exists, rcli merges its section using `<!-- rcli:start -->` / `<!-- rcli:end -->` markers. This allows other tools to coexist in the same file. Re-running `rcli init` replaces only the rcli section.

## Regeneration

All instruction files are regenerated on `rcli init`. It can also be manually regenerated by running init again.
